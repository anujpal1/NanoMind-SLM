# NanoMind-SLM Dataset Plan

## Day 2 objective

Verify the dataset source and design a memory-safe data pipeline before
downloading or processing training data.

## Proposed source

- Hugging Face dataset ID: `codeparrot/codeparrot-clean`
- Domain: Python source code
- Intended split: `train`
- Access method: Hugging Face Datasets streaming
- Primary text field: `content`

## Why this dataset

The dataset contains Python files collected from GitHub. It has already
undergone exact deduplication and basic quality filtering. This makes it a
better starting point than an unfiltered, multi-language GitHub dump.

## Fields to verify

- `content`: Python source text used for tokenizer and model training
- `repo_name`: source repository name
- `path`: original file path
- `license`: detected source-file license
- `size`: file size
- `hash`: content identifier
- `line_mean`: average line length
- `line_max`: maximum line length
- `alpha_frac`: alphanumeric-character fraction
- `autogenerated`: automatic-generation indicator

## Safety constraints

1. Never load the complete dataset into RAM.
2. Always use `streaming=True`.
3. Inspect only a small number of records during verification.
4. Set explicit sample or token limits for every experiment.
5. Do not save raw source files unless a later step explicitly requires it.
6. Never print complete source files during inspection.
7. Preserve license metadata during filtering.
8. Use deterministic seeds for shuffle buffers and sampling.

## Initial filtering plan

The quality-filtered experiment will consider:

- non-empty Python content;
- valid string content;
- reasonable minimum and maximum character lengths;
- removal of likely generated or vendored files;
- removal of unusually long lines;
- syntax-parsing checks where practical;
- license-aware filtering;
- exact-hash deduplication within our selected stream;
- protection against train/evaluation overlap.

## Planned experiments

### Experiment A: baseline subset

Use the existing CodeParrot Clean stream with only essential safety checks.

### Experiment B: quality-filtered subset

Apply additional project-level quality and syntax filters.

Both experiments should use comparable token budgets so that the comparison
measures data quality rather than dataset quantity.

## Planned data flow

Hugging Face stream
→ metadata validation
→ bounded quality filtering
→ tokenizer training sample
→ tokenization
→ fixed-length training sequences
